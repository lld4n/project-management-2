# MODEL DECISION

## Итоговое решение по модели

### Основная модель

- `gemma3:4b`

### Backup-модель

- `phi4-mini`

---

## Почему выбрана `gemma3:4b`

По итогам локального сравнения на `MacBook Air M1 16GB` модель `gemma3:4b` показала лучший суммарный баланс по критериям, которые важны именно для агентной системы:

- лучшее поведение на отказе при недостатке данных;
- лучшая дисциплина на первых grounded-кейсах;
- более адекватное понимание, когда нужен расчёт или инструмент;
- приемлемая скорость и работа в локальном `Ollama` runtime;
- меньше operational risk, чем у `qwen3:4b` в текущем стеке.

---

## Короткое сравнение кандидатов

### `gemma3:4b`

- лучший текущий кандидат;
- самый чистый отказ на unanswerable вопросах;
- лучше других понимает routing-логику;
- основной минус: строгий JSON без доп. hardening пока не держит идеально.

### `phi4-mini`

- хороший backup;
- приемлемое factual поведение;
- уступает по чистоте отказа и по tool-routing;
- тоже требует доп. настройки для строгого JSON.

### `llama3.2:3b`

- полезный baseline;
- проиграла по склонности к галлюцинациям;
- не подходит как основная модель агента.

### `qwen3:4b`

- потенциально сильная модель;
- но в текущей связке с `Ollama` показала нестабильное output behavior;
- не фиксируется как основная модель из-за operational risk.

---

## Практический вывод

Для дальнейшего проектирования агентной системы принимаем:

- основная LLM: `gemma3:4b`
- fallback / backup: `phi4-mini`

Все следующие решения по prompts, skills, memory и orchestration делаем под `gemma3:4b`.

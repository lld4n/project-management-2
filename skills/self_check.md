# Skill: self_check

## Назначение

Используется verifier-ом и report agent-ом перед финальным выводом.

## Checklist

1. Есть ли у каждого утверждения опора в данных?
2. Нет ли психологических или причинных скачков?
3. Не подменён ли расчёт предположением?
4. Есть ли ограничения данных, которые нужно явно назвать?
5. Не пытается ли ответ звучать увереннее, чем позволяют факты?

## Если ответ на один из вопросов плохой

- смягчить формулировку;
- отклонить claim;
- запросить дополнительный factual result.

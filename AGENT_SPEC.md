# AGENT SPEC

## Цель

Построить локальную мультиагентную систему анализа музыкальной истории пользователя на базе `gemma3:4b`.

Система должна:

- отвечать на вопросы по музыкальной истории;
- строить аналитические summaries и weekly digests;
- выявлять паттерны, тренды и сдвиги вкуса;
- не выдумывать факты при отсутствии подтверждения в данных;
- уметь объяснять, на каких фактах основан вывод.

---

## 1. Базовый контекст проекта

### Данные

Источник данных:

- `data/raw`

Формат:

- JSON-снимки истории;
- внутри есть дни истории;
- у дней есть recommendation items;
- item содержит `context` и список `tracks`;
- треки содержат метаданные: артист, альбом, жанр, длительность и другие поля;
- встречаются неполные записи с `error: no-rights`.

### Выбранная модель

- основная модель: `gemma3:4b`
- backup: `phi4-mini`

---

## 2. Что должна делать система

### Основные сценарии

1. Отвечать на factual-вопросы по музыкальной истории.
2. Давать краткие аналитические summaries по периоду.
3. Выявлять:
   - доминирующие жанры;
   - устойчивых артистов;
   - повторяющиеся треки;
   - изменения между неделями;
   - краткосрочные всплески и устойчивые тренды.
4. Готовить weekly digest / insight report.
5. Отказываться от ответа, если вопрос не покрывается данными.

### Что система не должна делать

- делать психологические выводы о пользователе;
- придумывать причины изменения настроения;
- называть "любимый" трек или артиста без явно определённого критерия;
- выдавать неподтверждённые причинные связи;
- смешивать факты и интерпретации без маркировки.

---

## 3. Типы запросов

### Factual

Примеры:

- какой жанр встречался чаще всего;
- какие артисты чаще всего встречались;
- какой контекст рекомендаций доминировал.

### Comparative

Примеры:

- что изменилось между февралём и апрелем;
- какие артисты появились в конце периода, но не были заметны в начале.

### Trend / insight

Примеры:

- какие паттерны выглядят устойчивыми;
- что было краткосрочным всплеском;
- какие жанры доминировали по неделям.

### Report

Примеры:

- собери weekly digest;
- подготовь краткий аналитический отчёт по истории.

---

## 4. Нефункциональные требования

- локальный запуск на `MacBook Air M1 16GB`;
- основная модель должна работать через `Ollama`;
- ответы должны быть grounded;
- система должна быть пригодна для агентного orchestration;
- итоговый вывод должен быть воспроизводимым;
- для важных выводов должны быть доступны факты / evidence.

---

## 5. Требования к качеству ответа

### Обязательно

- разделять факт и интерпретацию;
- указывать ограничения данных;
- не делать выводов сверх доступных фактов;
- использовать инструменты / расчёты там, где без них нельзя ответить честно.

### Желательно

- короткие, структурированные ответы;
- единый стиль аналитических summaries;
- минимизация лишней болтовни;
- устойчивое поведение на JSON / schema tasks.

---
